# BondKonnect: User Navigation Workflows

Step-by-step guides for the AI to assist users in maneuvering the platform.

## 1. Finding & Analyzing Bonds

**Goal:** User wants to see available bonds and their returns.
1. Direct user to **Bond Statistics** (`/dashboard/apps/bond-stats`).
2. Advise using the **Search bar** to filter for specific issues like "IFB".
3. Explain the **Customize View** button if they want to see "Risk Budgeting Indicators".
4. Suggest using the **Calculator** (`/dashboard/apps/analysis`) for specific YTM calculations.

## 2. Managing Your Portfolio

**Goal:** User wants to track their performance.
1. Direct user to **Portfolio Analytics** (`/dashboard/apps/portfolio-analytics`).
2. Instruct the user to **Select a Portfolio** from the dropdown menu.
3. Explain the **PnL Summary Cards**:
    - **Total Portfolio Value:** Your current investment worth.
    - **Total Realized P&L:** Profit/Loss from bonds already sold.
    - **Total Unrealized P&L:** Current Profit/Loss from bonds you still hold.
4. Point out the **Institutional Terminal** banner at the top to reinforce platform branding.

## 3. Funding Your Account (M-Pesa)

**Goal:** User wants to deposit money via M-Pesa.
1. Direct user to **Billing & Payments** (`/dashboard/apps/billing`).
2. Scroll to the **Payment Methods** section.
3. Click **Add Method** -> Select **Mobile Payment**.
4. Enter the **Phone Number** (+254...) to link a **Mobile Node**.
5. Once linked, explain that they will receive an **STK Push** on their phone to authorize payments.

## 4. Getting Support

**Goal:** User needs help or has a question.
1. Suggest visiting the **FAQ** (`/dashboard/apps/faq`) for common issues.
2. Direct user to **Messages** (`/dashboard/apps/messages`) to speak with a broker.
3. Mention the **Help Center** (`/dashboard/apps/help`) for platform-related documentation.
4. Advise checking **Activity Logs** (`/dashboard/apps/activity-logs`) to see a history of their actions.

## 5. Subscription Management

**Goal:** User wants to upgrade their account.
1. Direct user to **Pricing** (`/dashboard/apps/pricing`) or **Billing** (`/dashboard/apps/billing`).
2. Click **Modify Subscription** in the **Current Plan** card.
3. Explain the benefits of the **Institutional Terminal** plan for advanced trading.
